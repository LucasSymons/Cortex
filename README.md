# Cortex

> Your AI profile, synced everywhere.

Cortex is a Claude Code / Cowork plugin that keeps your AI persona, memory, and accumulated knowledge in sync across devices and AI platforms via Git.

## What it does

- **Portable identity** - your persona, rules, and memory follow you across devices and reinstalls
- **Shared knowledge** - lessons learned in one project feed back into your top-level memory
- **Cross-AI ready** - unified profile format with per-platform adapters (Claude, Gemini, ChatGPT)
- **Secure by default** - private repo, no PII in memory files, credentials in the OS keychain (with an encrypted-file fallback)
- **Zero friction** - auto-syncs on session handoff; manual sync available any time

## Installation

Cortex installs as a Claude Code plugin. The plugin bundles a small launcher
that, on first run, downloads the prebuilt `cortex-git-server` binary for your
platform from the matching GitHub release and verifies its SHA-256 before
running it — nothing to compile.

**Prerequisites:** `curl`, `tar`, and `sha256sum` (or `shasum`) on your `PATH`.
On Windows, run Claude Code under WSL.

```shell
/plugin marketplace add LucasSymons/Cortex
/plugin install cortex@cortex
```

On the next session the binary is fetched into the plugin's data directory and
cached for subsequent runs, then run `/setup` to build your profile.

Developers running from a local checkout can build the server with `make build`;
the launcher prefers a local build when present. See
[Running from a local checkout](CONTRIBUTING.md#running-from-a-local-checkout).

> **Surfaces:** the steps above are for **Claude Code CLI**. For the **Claude
> Desktop app and Cowork**, Cortex ships a `.mcpb` desktop-extension bundle that
> runs `cortex-git` as a local MCP server, with your PAT supplied through Claude's
> config UI - see [Cowork / Claude Desktop](docs/usage.md#cowork--claude-desktop-mcpb-bundle).
> Browser and mobile have no local runtime and are out of scope.

## Quick start

```
/setup            First-run: builds your profile and initialises the Git repo
/sync-profile     Manually sync memory to Git
/restore-profile  New device: clone your profile and place CLAUDE.md
/promote-lessons  Promote project lessons to top-level memory
```

`/setup` includes one manual step - creating an empty private repo in your Git
host's web UI. See the [usage guide](docs/usage.md) for the full walkthrough and
troubleshooting.

## Project structure

```
cortex/
├── .claude-plugin/plugin.json   Plugin manifest
├── skills/                      Claude skills (setup, sync, restore, promote)
├── mcp/git-server/              Go MCP server for Git operations
│   ├── cmd/server/main.go       Entry point
│   ├── internal/git/            go-git wrapper
│   ├── internal/keychain/       credential storage (keychain + encrypted-file fallback)
│   └── Makefile                 Build targets
├── profile-template/            Template files generated by setup skill
│   ├── memory/                  Memory file templates
│   └── adapters/                Platform adapter templates
└── .mcp.json                    MCP server registration
```

## Building the MCP server

See [Running from a local checkout](CONTRIBUTING.md#running-from-a-local-checkout)
in CONTRIBUTING for the build (`make build` / `make build-all`) and run steps.

## Tech stack

- **MCP server:** Go 1.26+ (build pinned to 1.26.4)
- **Git:** `go-git` (pure Go, no system git dependency)
- **Credentials:** `zalando/go-keyring` (OS keychain), with an AES-256-GCM encrypted-file fallback on headless/WSL platforms
- **Transport:** HTTPS + PAT (no SSH)
