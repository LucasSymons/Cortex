# Cortex Memory System

Persistent knowledge that travels across sessions, devices, and AI platforms.

## Files

| File | Purpose |
|---|---|
| `active.md` | Current sprint/project context - what's in flight right now |
| `systems.md` | Systems you work with - contacts, quirks, useful links, incident patterns |
| `people.md` | Team members and stakeholders - what they own, when to loop them in |
| `lessons.md` | Learned patterns, gotchas, architectural decisions and rationale |

## Conventions

- **Format:** Each entry is a short heading + bullet points. No padding.
- **Dating:** Prefix entries with the date added: `(YYYY-MM-DD)`.
- **Saving:** Say "save to memory" - the AI will add to the right file and confirm.
- **Proactive prompts:** The AI should suggest saving when a new lesson, quirk, decision, or contact surfaces.
- **Sync:** Memory is synced to Git automatically on session handoff, or manually via `/sync-profile`.

## What triggers a memory save

- A system quirk or workaround is discovered
- An architectural or approach decision is made
- A new contact, owner, or escalation path is identified
- Something breaks in an unexpected way (lesson learned)
- A task completes and the how/why is worth keeping
- A person's role or responsibility is clarified or corrected
- A process or policy detail is confirmed from an authoritative source
